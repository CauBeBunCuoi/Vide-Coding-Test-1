# Pipeline: Init Worktrees

Run this pipeline AFTER `init-team` has completed and the team is ready.
It creates an isolated git worktree per agent and updates each real agent file
so teammates know their exact working directory.

Follow every step in order. Do not skip or assume — execute each step explicitly.

---

## Hard Rules

- Never modify `.example.md` files — update real agent files only
- Real agent files follow `<project-kebab>-agent_<role>.md` (no `.example` in name)
- Worktrees are created under `.worktrees/` in the project root
- Each agent owns exactly one worktree — no sharing

---

## Step 1 — Confirm Real Agent Files Exist

Before creating worktrees, confirm `init-team` has already run and real agent files exist:

```bash
ls .claude/agents/
```

Real files must exist (e.g. `vide-coding-test-1-agent_frontend.md`).
If only `.example.md` files are present, STOP — run `init-team` first.

---

## Step 2 — Check For Existing Worktrees

```bash
git worktree list
```

If `.worktrees/taskflow-fe`, `.worktrees/taskflow-be`, or `.worktrees/taskflow-test`
already exist, STOP and report to the user — do not recreate over existing work.

---

## Step 3 — Create Worktrees

```bash
git worktree add .worktrees/taskflow-fe -b feat/frontend
git worktree add .worktrees/taskflow-be -b feat/backend
git worktree add .worktrees/taskflow-test -b feat/test
```

Verify all three were created:

```bash
git worktree list
```

---

## Step 4 — Update Agent Working Directory In Real Agent Files

For each real agent file, update the `## Working Directory` section to point
to the agent's subproject folder inside their worktree.

**Frontend agent** (`<project-kebab>-agent_frontend.md`):
```markdown
## Working Directory
Your workspace is `.worktrees/taskflow-fe/taskflow-fe/` (relative to project root).
- Treat all file paths as relative to `.worktrees/taskflow-fe/taskflow-fe/`
- On every new task, read `.worktrees/taskflow-fe/taskflow-fe/CLAUDE.md` first for project context
- You are on branch `feat/frontend` — commit your work to this branch only
```

**Backend agent** (`<project-kebab>-agent_backend.md`):
```markdown
## Working Directory
Your workspace is `.worktrees/taskflow-be/taskflow-be/` (relative to project root).
- Treat all file paths as relative to `.worktrees/taskflow-be/taskflow-be/`
- On every new task, read `.worktrees/taskflow-be/taskflow-be/CLAUDE.md` first for project context
- You are on branch `feat/backend` — commit your work to this branch only
```

**Tester agent** (`<project-kebab>-agent_tester.md`):
```markdown
## Working Directory
Your workspace is `.worktrees/taskflow-test/taskflow-test/` (relative to project root).
- Treat all file paths as relative to `.worktrees/taskflow-test/taskflow-test/`
- On every new task, read `.worktrees/taskflow-test/taskflow-test/CLAUDE.md` first for project context
- You are on branch `feat/test` — commit your work to this branch only
```

---

## Step 5 — Re-spawn Teammates

In-process teammates persist within a session — they do not reload their agent file
automatically after it is updated on disk. A re-spawn is required.

### 5a — Shut down all teammates

Tell the lead:
```
Shut down all teammates gracefully.
```
Wait for all teammates to confirm shutdown before proceeding.

### 5b — Re-spawn the team

Tell the lead:
```
Spawn the full team again using the role files in .claude/agents/. Stand by for tasks.
```
Teammates now load the updated agent files with the correct worktree path and branch.

### 5c — Verify

Ask each teammate:
```
What is your current working directory and branch?
```

Expected responses:
- Teammate A: `.worktrees/taskflow-fe/taskflow-fe/` on `feat/frontend`
- Teammate B: `.worktrees/taskflow-be/taskflow-be/` on `feat/backend`
- Teammate C: `.worktrees/taskflow-test/taskflow-test/` on `feat/test`

If any teammate reports the wrong path, re-send their spawn prompt and verify again.

---

## Step 6 — Report To User

```
Worktrees ready. Team re-spawned with updated context.

| Agent      | Branch          | Workspace                                    |
|------------|-----------------|----------------------------------------------|
| Teammate A | feat/frontend   | .worktrees/taskflow-fe/taskflow-fe/          |
| Teammate B | feat/backend    | .worktrees/taskflow-be/taskflow-be/          |
| Teammate C | feat/test       | .worktrees/taskflow-test/taskflow-test/      |

Team is ready for tasks.
```

---

## Teardown (run when merging is complete)

After all branches are merged into main, remove worktrees:

```bash
git worktree remove .worktrees/taskflow-fe
git worktree remove .worktrees/taskflow-be
git worktrees remove .worktrees/taskflow-test
git branch -d feat/frontend feat/backend feat/test
```

---

## Notes

- Worktrees are full repo checkouts on isolated branches — agents cannot conflict
- Each agent reads their subproject CLAUDE.md from inside their own worktree
- The original `taskflow-fe/`, `taskflow-be/`, `taskflow-test/` in root remain on main — untouched until merge
- Re-running this pipeline on an existing worktree setup will STOP at Step 2 to protect in-progress work
