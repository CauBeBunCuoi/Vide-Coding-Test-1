# Agent Monitoring — Windows Setup

Claude Code agent teams run in `in-process` mode on Windows.
Split-pane (tmux) mode is not supported on Windows Terminal — officially documented as unsupported.

## Settings

`teammateMode` is set in `.claude/settings.json`:
```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  },
  "teammateMode": "in-process"
}
```

## How In-Process Mode Works

All teammates run inside your main terminal. You navigate between them using keyboard shortcuts:

| Key | Action |
|---|---|
| `Shift+Down` | Cycle through Lead → Teammate A → Teammate B → Teammate C → Lead |
| `Enter` | Enter that teammate's session |
| `Escape` | Interrupt their current turn |
| `Ctrl+T` | Toggle shared task list (pending / in-progress / completed) |
| Type | Send a message directly to the current teammate |

You can inspect what any teammate is doing mid-task by cycling to them.
Automatic notifications arrive when teammates finish — no polling needed.

## Spawning the Team

Ask Claude in natural language from your terminal:
```
Create an agent team with a frontend developer, backend developer, and tester
using the role files in .claude/agents/. Stand by for tasks.
```

Claude Code handles spawning, task list setup, and inter-agent messaging automatically.
Do NOT pre-author team config files — Claude Code generates and manages them.

## Notes

- tmux split-pane mode is unsupported on Windows Terminal (documented limitation)
- If running Claude Code inside WSL (Linux), `teammateMode: "auto"` or `"tmux"` would work — but project files must live in WSL, not on a Windows drive
- The old manual tmux bootstrap is archived in `tmux-agents-setup-old.md`
