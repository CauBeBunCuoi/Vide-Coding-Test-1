# How to Implement a Feature

1. Read `references/user-flows.md` to understand the user journey this feature belongs to
2. Identify all screens and components involved
3. Implement or update the required components (see `create-component` skill)
4. Wire up routing and navigation between screens
5. Connect to the API layer — add or update the relevant API call
6. Handle loading, error, and empty states
7. Test the full flow manually against the acceptance criteria

- `references/user-flows.md` — key user journeys through the application
