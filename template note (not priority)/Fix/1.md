## Why This Change Is Needed

The current setup has three problems:

### Problem 1: `.claude/skills/` files are not discoverable

Claude Code's official skill system requires this exact structure:
```
.claude/skills/<skill-name>/SKILL.md
```

The current files like `.claude/skills/add-endpoint.md` will NEVER be auto-discovered by Claude Code because:
- The file must be named `SKILL.md` (not `add-endpoint.md`)
- It must be inside a subdirectory (not loose in `skills/`)

Even though the current files have YAML frontmatter, Claude Code ignores them because the file structure is wrong.

### Problem 2: `.claude/context/` does not exist in Claude Code

Claude Code has only two official mechanisms:
1. `.claude/rules/` — files here auto-load every session, same priority as CLAUDE.md. No import needed.
2. `.claude/skills/<name>/SKILL.md` — files here are auto-discovered. Only the frontmatter (~100 tokens) loads at startup. The full body loads only when Claude decides the skill is relevant. Supporting files in the skill directory load only when needed.

There is no `.claude/context/` in the official system. Content currently in `context/` must be relocated:
- If it's always needed → move to `.claude/rules/`
- If it's only needed during a specific task → move into a skill's directory as a reference file

### Problem 3: `CLAUDE.md` uses `@import` for everything

Since `.claude/rules/` auto-loads and `.claude/skills/` auto-discovers, the `CLAUDE.md` in each workspace does not need `@import` for any of these. CLAUDE.md should only contain the project overview and any always-on context that doesn't fit in a rules file.

## How Claude Code's Progressive Loading Works

When Claude Code starts a session, it loads in 3 tiers:

**Tier 1 — Always loaded (system prompt):**
- `CLAUDE.md` (full content)
- All `.md` files in `.claude/rules/` (full content)
- YAML frontmatter only (name + description) from every `SKILL.md` found in `.claude/skills/*/`

**Tier 2 — Loaded on demand (when Claude decides the skill is relevant):**
- The full body of the matching `SKILL.md`

**Tier 3 — Loaded only when the skill explicitly needs it:**
- Reference files, templates, scripts inside the skill's directory

This means you can have 30 skills installed and only pay ~100 tokens per skill at startup. The full instructions only load when actually needed.
